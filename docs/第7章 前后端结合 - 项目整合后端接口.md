### 笔记
- #NOTE
	- 如何在 vuecli3 中管理环境变量？
		- 在项目下新建 env.local，并添加以 VUE_APP_ 前缀开头的变量
	- 使用 axios 设置 base url 和 interceptor
		- 使用两者的目的都是为了简化后续请求 api 的写法
		```javascript
		// 设置前
		axios.get(`http://apis.imooc.com/api/columns?icode=${ICODE}`).then(res => {
			console.log(res.data);
		})
		// 设置后
		axios.get(`/columns`).then(res => {
			console.log(res.data);
		})
		```

- [x] 修复 columnDetail 页面刷新时页面无内容
	- 预期：刷新页面，触发 onMounted，向服务器发送请求获取数据，接收数据后触发 computed 渲染页面
	- 实际：刷新页面，onMounted 未执行，未向服务器发送请求，访问了 undefined 上属性而报错
	![](attachments/columnDetail-bug.png)
	- 原因：当 find 未匹配到符合条件的值时，返回 undefined，导致后续访问 column 上一系列属性 (column 值为 undefined)时报错
	- 总结：computed 先于 mounted 执行
	- 补充：[Vue中的computed是在生命周期的哪个阶段执行的？ - SegmentFault 思否](https://segmentfault.com/q/1010000010364198)

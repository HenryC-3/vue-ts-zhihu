# vscode-drawio
- 问题
  - 使用指令 `drawio: create new drawio diagram` 时，只能新建 `.drawio` 格式文件，而我需要 `.drawio.svg`
  - 我需要执行默认文件夹为 `attachement`
  - 快捷键冲突，部分 vscode 快捷键无法在 drawio editor 中使用
  - mindmap 不会随着分支增加自动排列节点，如果节点过多，会挤在一起
  - ~~图像模糊~~，设置为 `.svg`
- mermaid vs drawio
  - mermaid 的缺点在于，随着绘制的图像逻辑逐渐复杂，mermaid 代码会越来越难看且难以修改。这点在绘制流程图时尤为明显。mermaid 更适合绘制时序图、饼状图、甘特图这类逻辑简单、偏数据驱动的图形，配合 code snippet 能够快速绘制样式统一的图形。而且 mermaid 的图形在 preview 时随着根据页面明暗、宽窄对图像样式进行动态调整
  - drawio 是可视化编辑器，创建、删除图形的成本很低。绘制时拖拽即可，非常适合绘制复杂流程图。但绘制时序图时，随着时序图内容增多，需要手动调整时序图图形，非常麻烦
  - drawio 添加 vscode drawio mermaid plugin 后，也支持绘制 mermaid。并且可以作为 svg 图片导出，但是会丧失动态调整的功能。但这也有一个好处，就是原来 markdown 文件中的一大段 mermaid 代码，缩小成了一句 `![alt](https://link)`，简洁不少
  - 所以目前的方案是
    - 在 drawio 中绘制所有图形，包括 mermaid
    - 绘制复杂流程图时，使用 drawio 拖拽绘制
    - 绘制时序图、饼状图、甘特图时，使用 mermaid 在 drawio 中绘制
    - 注意，在 drawio 中使用 mermaid 绘制图形时，如果图形过小，需要手动调整到合适大小，详细示例参考 [[vscode-drawio-mermaid-plugin]]
